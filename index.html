Instructions Of Use:

Download the font files and put them in this path:

./assets/fonts/

- Should add

lang="ar"

property to each tag or container has an arabic text.

Important information:

The font used is the same font used in UI/UX design, but two copies of the font file were created to solve a technical problem related to displaying the text correctly.

Problem Description:

When using a local font file, the fonts appear vertically incorrect by an estimated 2px or more, which affects the visual alignment within interface elements such as (buttons, input fields, headings), while this problem does not appear when using the font from external providers such as Google Fonts.

The problem is due to a difference in Font Metrics between the local version of the font and the modified version from providers such as Google Fonts, as some local fonts are not fully optimized in terms of ascent, descent, line-gap, which leads to a difference in the vertical positioning of the text (Vertical Offset).

To solve this problem, should modify:

- Line Height.

- Letter positioning (Vertical Alignment).

- Internal text spacing (Font Metrics).

By using the following CSS properties when importing the font:

ascent-override

descent-override

line-gap-override

* These properties only work when importing the font via @font-face!

Since the two languages ​​differ in the way they display text, the heights must be set for each language separately and with different numbers. Since the adjustment must be done through the CSS properties mentioned above, which only work through @font-face, it was necessary to create two separate font files for each language in order to set the heights correctly.

Example of the problem:

After readjusting:

Final solution:

/* ---------- Font Setup ---------- */
/* English version */
@font-face {
    font-family: 'mainFont-en';
    src: url('../fonts/mainFont-en.woff2') format('woff2');
    font-style: normal;
    font-display: swap;

    /* Font Line Height Setup */
    /* 
        IMPORTANT:
            These Metrics Are Calibrated Specifically For This Font !
            Changing The Font File Requires Recalibration.
    */
    ascent-override: 87%;
    descent-override: 13%;
    line-gap-override: 10%;
}

/* Applay English Font */
:lang(en), html {
    font-family: 'mainFont-en';
}

/* Arabic version */
@font-face {
    font-family: 'mainFont-ar';
    src: url('../fonts/mainFont-ar.woff2') format('woff2');
    font-style: normal;
    font-display: swap;

    /* Font Line Height Setup */
    /* 
        IMPORTANT:
            These Metrics Are Calibrated Specifically For This Font !
            Changing The Font File Requires Recalibration.
    */
    ascent-override: 80%;
    descent-override: 24%;
    line-gap-override: 32%;
}

/* Applay Arabic Font */
:lang(ar) {
    font-family: 'mainFont-ar';
}

* The (html) tag was added in english activation because english is the default language.

IMPORTANT:

These Metrics Are Calibrated Specifically For This Font !

Changing The Font File Requires Recalibration.

تعليمات الاستخدام:

- حمل ملفات الخط وضعها في هذا المسار:

./assets/fonts/

- يجب اضافة الخاصية

lang="ar"

الى كل tag او container يحتوي على نصوص عربية.

معلومات مهمة:

الخط المستخدم هو نفس الخط المستخدم في التصميم، ولكن تم إنشاء نسختين من ملف الخط وذلك لحل مشكلة تقنية تتعلق بعرض النصوص بشكل صحيح.

توصيف المشكلة:

عند استخدام ملف الخط المحلي تظهر الخطوط بشكل خاطئ عمودياً يقدر بـ2px او اكثر، مما يؤثر على المحاذاة البصرية داخل عناصر الواجهة مثل (الأزرار، حقول الادخال، العناوين)، بينما هذه المشكلة لا تظهر عند استخدام الخط من مزودين خارجيين مثل Google Fonts.

سبب المشكلة يعود الى اختلاف في Font Metrics بين النسخة المحلية من الخط والنسخة المجهزة والمعدلة من مزودين مثل Google Fonts، حيث أن بعض الخطوط المحلية لا تكون محسنة بالكامل من ناحية ascent، descent، line-gap، مما يؤدي الى اختلاف في تموضع النص عمودياً (Vertical Offset).

ولحل هذه المشكلة يجب ضبط:

- ارتفاع السطر (Line Height).

- تموضع الحروف عمودياً (Vertical Alignment).

- المسافات الداخلية للنصوص (Font Metrics).

عن طريق خصائص الـCSS التالية عند استدعاء الخط:

ascent-override

descent-override

line-gap-override

* هذه الخصائص تعمل فقط عند استدعاء الخط عبر خاصية @font-face فقط!

وبما ان اللغتين مختلفتين في طريقة عرض النصوص يجب ضبط الارتفاعات لكل لغة بشكل منفصل وبأرقام مختلفة وبما ان الضبط لابد ان يتم عن طريق خصائص الـCSS المذكورة بالاعلى والتي لا تعمل إلا عن طريق @font-face كان لابد من عمل ملفين منفصلين من الخط لكل لغة لكي يتم ضبط الارتفاعات بالشكل الصحيح.

Instructions of measurements:

- The Width And Height Are Not Fixed, You Can Use The Measurements As Needed.

- The Default Measurements Is 24px For Width And 24px For Height.

- Please Use The Same Number For Width And Height, Except Some Icons Which Can Be Changed According To Their Design.

- In Main Buttons Which Contain This Arrows

Use This Measurements:

width="10" height="18"

For Example:

Button

<div class="BTN Primary-BTN">
    <p>Button</p>

    <div class="BTNicon">
        <svg width="10" height="18">
            <use href="./assets/icons.svg#right"><use/>
        </svg>
    </div>
</div>

- In Arrows Buttons Which Contain This Arrows

Use This Measurements:

width="16" height="16"

For Example:

<div class="BTN Arrows-BTN">
    <svg width="16" height="16">
        <use href="./assets/icons.svg#right"></use>
    </svg>
</div>

تعليمات القياسات:

- العرض والارتفاع غير ثابتين، يمكنك تغييرها حسب الحاجة.

- القياسات الافتراضية هي 24px للعرض و 24px للارتفاع.

- يرجى استخدام نفس الرقم للعرض والارتفاع، باستثناء بعض الرموز التي يمكن تغييرها وفقًا لتصميمها.

- في الازرار الرئيسية التي تحتوي على هذه الاسهم:

استخدم هذه المقاسات:

width="10" height="18"

مثال:

زر رئيسي

<div class="BTN Primary-BTN">
    <p class="arabic">زر رئيسي</p>

    <div class="BTNicon">
        <svg width="10" height="18">
            <use href="./assets/icons.svg#left"><use/>
        </svg>
    </div>
</div>

- في ازرار التنقل التي تحتوي على هذه الاسهم:

استخدم هذه المقاسات:

width="16" height="16"

مثال:

<div class="BTN Arrows-BTN">
